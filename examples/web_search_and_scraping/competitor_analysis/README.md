# Competitor Analysis Agent

An AI-powered competitive intelligence tool built with the **Upsonic AI Agent Framework** and **Firecrawl**. Point it at your competitors' websites and get a structured analysis report — features, pricing, positioning, and strategic insights — all generated automatically.

## What It Does

The agent performs a multi-step research workflow:

1. **Maps** each competitor's website to discover available pages (pricing, features, about, docs)
2. **Scrapes** the most relevant pages using Firecrawl's clean Markdown extraction
3. **Searches** the web when information isn't directly on the website
4. **Analyzes** everything with the LLM and produces a structured report
5. **Outputs** a typed Pydantic model with competitor profiles, comparison tables, and strategic insights

## Quick Start

### 1. Install Dependencies

```bash
cd examples/web_search_and_scraping/competitor_analysis

# With uv (recommended)
uv venv && source .venv/bin/activate
uv pip install -r requirements.txt

# With pip
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
```

### 2. Set Up API Keys

```bash
cp .env.example .env
```

Edit `.env` with your keys:

```bash
FIRECRAWL_API_KEY=fc-your-key-here   # Free tier at firecrawl.dev
ANTHROPIC_API_KEY=sk-your-key-here
```

### 3. Customize Your Competitors

Open `config.py` and update:

```python
COMPETITOR_URLS = [
    "https://competitor-one.com",
    "https://competitor-two.com",
    "https://competitor-three.com",
]

INDUSTRY = "Your Industry Here"

FOCUS_AREAS = [
    "Core features and capabilities",
    "Pricing model and tiers",
    "Target audience and positioning",
]
```

### 4. Run the Analysis

```bash
python main.py
```

The report is saved as both `report.json` (structured) and `example_report.md` (readable).

> 📄 **Want to see the output before running?** Check out [`example_report.md`](example_report.md) for a full sample report generated by this agent.

## How It Works

### Firecrawl Tools Used

| Tool | Purpose |
|------|---------|
| `map_website` | Discovers all pages on a competitor's site so the agent knows what to scrape |
| `scrape_url` | Fetches individual pages as clean Markdown — strips nav, ads, boilerplate |
| `search_web` | Falls back to web search when info isn't on the competitor's site directly |

Only these three tools are enabled. The agent can't accidentally trigger crawls, batch operations, or extraction jobs.

### Agent Workflow

```
config.py (competitors)
    │
    ▼
┌─────────────────────────────┐
│  For each competitor URL:   │
│  1. map_website → page list │
│  2. scrape_url → key pages  │
│  3. search_web → fill gaps  │
└─────────────────────────────┘
    │
    ▼
┌─────────────────────────────┐
│  LLM Analysis:              │
│  • Build competitor profile │
│  • Compare features/pricing │
│  • Generate insights        │
└─────────────────────────────┘
    │
    ▼
CompetitiveAnalysisReport (Pydantic)
    │
    ├── Terminal output (formatted)
    └── report.json (structured)
```

### Structured Output

The agent returns a `CompetitiveAnalysisReport` Pydantic model, not free-form text. This means:

- **Type-safe fields** — access `result.competitors[0].pricing_model` directly
- **JSON export** — `result.model_dump()` for APIs, databases, or dashboards
- **Customizable schema** — edit `schemas.py` to extract different data points

## Customization Guide

### Analyze a Different Market

Edit `config.py`:

```python
COMPETITOR_URLS = [
    "https://stripe.com",
    "https://square.com",
    "https://adyen.com",
]
INDUSTRY = "Payment Processing"
FOCUS_AREAS = [
    "Supported payment methods",
    "Developer API quality",
    "Transaction fees and pricing",
    "Geographic coverage",
]
```

### Use a Different LLM

```python
# config.py
MODEL = "anthropic/claude-sonnet-4-5"   # Anthropic
MODEL = "openai/gpt-4o"                 # OpenAI (higher quality)
MODEL = "openai/gpt-4o-mini"            # OpenAI (faster/cheaper)
```

### Add Custom Fields to the Report

Edit `schemas.py` to add fields:

```python
class CompetitorProfile(BaseModel):
    # ... existing fields ...
    tech_stack: List[str] = Field(description="Technologies used")
    founded_year: Optional[int] = Field(description="Year the company was founded")
    funding_info: Optional[str] = Field(description="Known funding rounds")
```

The agent will automatically try to find this information during scraping.

### Adjust Scraping Depth

```python
# config.py
MAP_LIMIT = 100        # Discover more pages per site (uses more API calls)
SCRAPE_TIMEOUT = 60    # Allow slower pages to load
```

## Notes

- The analysis typically takes 2-5 minutes depending on the number of competitors and model speed
- Firecrawl's free tier provides 500 credits — enough for several full analyses
- The agent prioritizes pages with "pricing", "features", "about", and "product" in their URLs
- All data comes from actual website content — the LLM doesn't hallucinate pricing or features
- Output is saved as both formatted terminal output and structured JSON
